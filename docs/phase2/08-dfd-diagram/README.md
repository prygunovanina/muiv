# Формирование DFD-диаграммы верхнего уровня

На основе выполненной декомпозиции системы и анализа потоков данных сформирована диаграмма потоков данных верхнего уровня.

## Внешние сущности

### Пользователь
Сотрудник организации, взаимодействующий с системой для работы с документами

#### Действия
- Отправка запросов на создание, просмотр, редактирование документов
- Получение уведомлений о необходимости выполнения действий
- Просмотр отчетов

### Администратор
Технический специалист, управляющий настройками системы

#### Действия
- Настройка параметров системы
- Управление справочниками
- Контроль работы компонентов
- Получение информации о состоянии системы

### Внешние системы
Сторонние информационные системы, с которыми интегрируется система делопроизводства

#### Взаимодействие
- Отправка и получение данных о документах
- Синхронизация справочников
- Обмен событиями

### Почтовый сервер
Система электронной почты для отправки уведомлений пользователям

#### Функция
Доставка уведомлений адресатам

## Основные процессы системы

### Процесс 1.0 Аутентификация пользователя

#### Описание
Проверка учетных данных пользователя при входе в систему

#### Потоки данных
- Входные: логин и пароль от пользователя
- К БД пользователей: запрос проверки
- От БД пользователей: результат проверки
- Выходные: токен доступа пользователю

### Процесс 2.0 Регистрация документа

#### Описание
Создание новой карточки документа в системе

#### Потоки данных
- Входные: данные документа и файлы от пользователя
- К БД документов: метаданные документа
- К файловому хранилищу: файлы документа
- К журналу аудита: запись о создании
- Выходные: подтверждение создания пользователю

### Процесс 3.0 Маршрутизация документа

#### Описание
Организация движения документа по этапам согласования

#### Потоки данных
- Входные: решение по документу от пользователя
- К БД документов: обновление статуса
- К почтовому серверу: уведомления
- От почтового сервера к пользователю: письма с уведомлениями
- Выходные: задача на согласование пользователю

### Процесс 4.0 Поиск документа

#### Описание
Нахождение документов по заданным критериям

#### Потоки данных
- Входные: критерии поиска от пользователя
- К БД документов: поисковый запрос
- От БД документов: результаты поиска
- Выходные: список документов пользователю

### Процесс 5.0 Формирование отчета

#### Описание
Генерация аналитических отчетов о состоянии документооборота

#### Потоки данных
- Входные: параметры отчета от пользователя
- К БД документов: запрос данных
- К журналу аудита: запрос данных аудита
- Выходные: готовый отчет пользователю

### Процесс 6.0 Управление архивом

#### Описание
Перенос документов в архив и управление архивным хранением

#### Потоки данных
- К БД документов: обновление статусов
- К файловому хранилищу: перенос файлов

## Хранилища данных

### База данных документов
Реляционное хранилище метаданных документов

#### Содержимое
- Карточки документов
- Реквизиты
- Связи
- Статусы

### Файловое хранилище
Объектное хранилище файлов документов

#### Операции
- Запись и чтение файлов
- Доступ для процессов регистрации и поиска

### База данных пользователей
Хранилище учетных записей пользователей

#### Содержимое
- Учетные записи
- Роли
- Права доступа

### Журнал аудита
Хранилище записей о всех операциях в системе

#### Использование
- Фиксация действий всеми процессами
- Источник данных для отчетности

## Потоки данных между элементами

### Взаимодействие с внешними системами
- От внешних систем к процессу 2.0: данные для импорта
- От процесса 2.0 к внешним системам: подтверждение импорта
- От администратора к системе: настройки и параметры

Диаграмма демонстрирует основные потоки информации в системе и взаимосвязи между компонентами, обеспечивая понимание общей структуры обработки данных.
