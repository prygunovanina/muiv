# Разработка сценариев тестирования

Для обеспечения качества разработанной информационной системы делопроизводства разработаны сценарии тестирования, охватывающие основные функциональные возможности и критичные пути использования.

## Сценарий 1: Создание и регистрация документа

### Предусловия
Пользователь авторизован в системе, имеет права на создание документов

### Шаги теста
1. Открыть форму создания нового документа
2. Заполнить обязательные реквизиты: название, тип документа, подразделение
3. Прикрепить файл документа в формате DOCX размером до 10 МБ
4. Нажать кнопку "Зарегистрировать"

### Ожидаемый результат
- Документ создан в системе с присвоением уникального регистрационного номера
- Файл сохранен в хранилище и доступен для скачивания
- Отображено сообщение об успешной регистрации
- Документ появился в списке документов пользователя
- Запись о создании документа присутствует в журнале аудита

### Критерии успеха
- Время создания документа не превышает 3 секунд
- Регистрационный номер соответствует установленному формату
- Все метаданные корректно сохранены

## Сценарий 2: Поиск документа по реквизитам

### Предусловия
В системе присутствуют тестовые документы с известными реквизитами

### Шаги теста
1. Открыть страницу поиска документов
2. Ввести критерии поиска: номер документа, диапазон дат, подразделение
3. Нажать кнопку "Найти"
4. Проверить результаты поиска

### Ожидаемый результат
- Отображен список документов, соответствующих критериям поиска
- Документы, не соответствующие критериям, отсутствуют в результатах
- Для каждого найденного документа отображены основные реквизиты
- Доступны действия по открытию и просмотру найденных документов

### Критерии успеха
- Время выполнения поиска не превышает 2 секунд для базы из 10000 документов
- Полнота результатов - найдены все документы, удовлетворяющие критериям
- Точность результатов - отсутствуют ложные срабатывания

## Сценарий 3: Согласование документа

### Предусловия
Документ создан и зарегистрирован, настроен маршрут согласования с двумя этапами

### Шаги теста
1. Отправить документ на согласование
2. Войти в систему под учетной записью первого согласующего
3. Открыть задачу на согласование
4. Просмотреть документ
5. Принять положительное решение с комментарием
6. Войти в систему под учетной записью второго согласующего
7. Открыть задачу на согласование
8. Принять положительное решение
9. Проверить статус документа

### Ожидаемый результат
- После отправки на согласование статус документа изменился на "На согласовании"
- Первому согласующему создана задача и отправлено уведомление
- После решения первого согласующего задача закрыта, решение зафиксировано
- Второму согласующему создана задача после завершения первого этапа
- После положительных решений всех участников статус документа изменился на "Утвержден"
- Автору отправлено уведомление о завершении согласования

### Критерии успеха
- Уведомления доставлены в течение 1 минуты после создания задачи
- История согласования содержит все решения с комментариями и временными метками
- Документ не может быть изменен в процессе согласования

## Сценарий 4: Полнотекстовый поиск документов

### Предусловия
В системе проиндексированы документы с известным содержимым

### Шаги теста
1. Открыть форму поиска
2. Ввести поисковый запрос, содержащий слово из текста документа в различных морфологических формах
3. Выполнить поиск
4. Проверить релевантность результатов

### Ожидаемый результат
- Найдены документы, содержащие искомое слово в любой морфологической форме
- Результаты отсортированы по релевантности
- Выделены фрагменты текста с найденными словами
- Указано количество вхождений в каждом документе

### Критерии успеха
- Поиск учитывает морфологию русского языка
- Время поиска не превышает 1 секунды для базы из 50000 документов
- Полнота поиска - найдены все релевантные документы

## Сценарий 5: Формирование отчета о количестве документов

### Предусловия
В системе имеются документы различных типов за разные периоды

### Шаги теста
1. Открыть раздел отчетов
2. Выбрать тип отчета "Количество документов по типам"
3. Указать период формирования отчета
4. Нажать кнопку "Сформировать"
5. Проверить содержание отчета

### Ожидаемый результат
- Сформирован отчет в табличном виде
- В отчете отображены все типы документов с количеством по каждому типу
- Итоговая сумма соответствует общему количеству документов за период
- Доступен экспорт отчета в форматы XLSX и PDF

### Критерии успеха
- Время формирования отчета не превышает 5 секунд
- Данные в отчете соответствуют фактическому количеству документов
- Форматирование отчета соответствует корпоративным стандартам

## Сценарий 6: Массовое изменение статуса документов

### Предусловия
В системе имеется группа документов, готовых для переноса в архив

### Шаги теста
1. Найти документы по критерию "дата создания старше 3 лет"
2. Выбрать все найденные документы
3. Выбрать действие "Перенести в архив"
4. Подтвердить операцию
5. Дождаться завершения операции

### Ожидаемый результат
- Статус всех выбранных документов изменен на "В архиве"
- Файлы документов перемещены в архивное хранилище
- Отображен прогресс выполнения операции
- По завершении показано количество обработанных документов
- В журнале аудита зафиксирована операция

### Критерии успеха
- Операция обрабатывает не менее 100 документов в минуту
- При сбое обработки одного документа операция продолжается для остальных
- Предоставлен отчет о документах, обработка которых завершилась с ошибкой

## Сценарий 7: Восстановление документа из архива

### Предусловия
Документ находится в статусе "В архиве"

### Шаги теста
1. Найти архивный документ
2. Открыть карточку документа
3. Выбрать действие "Восстановить из архива"
4. Подтвердить операцию

### Ожидаемый результат
- Статус документа изменен на "Зарегистрирован"
- Файл документа доступен для просмотра и скачивания
- Документ доступен для редактирования и отправки на согласование
- Операция восстановления зафиксирована в истории документа

### Критерии успеха
- Время восстановления не превышает 10 секунд
- Все метаданные и файлы документа полностью восстановлены
- Сохранена история всех операций с документом

## Критерии успешного функционирования системы

### Производительность
- Время отклика на запросы пользователя не более 3 секунд в 95% случаев
- Система поддерживает одновременную работу не менее 200 пользователей
- Пропускная способность не менее 1000 операций с документами в час

### Доступность
- Время доступности системы не менее 99% в месяц
- Время восстановления после сбоя не более 30 минут

### Безопасность
- Все операции пользователей фиксируются в журнале аудита
- Несанкционированный доступ к документам предотвращен
- Данные при передаче зашифрованы

### Масштабируемость
- Система сохраняет производительность при росте базы документов до 1 миллиона
- Добавление новых пользователей не требует модификации архитектуры
